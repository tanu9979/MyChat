# Real-Time Chat Application

A production-ready real-time chat application built for the Tars Full Stack Engineer Internship assignment.

## 🚀 Tech Stack

- **Frontend:** Next.js 15 (App Router), TypeScript, Tailwind CSS
- **Backend:** Convex (real-time database)
- **Authentication:** Clerk
- **Deployment:** Vercel

## 🛠️ Setup Instructions

### Prerequisites
- Node.js 18+ installed
- npm or yarn package manager
- Clerk account (free tier)
- Convex account (free tier)

### 1. Clone the Repository
```bash
git clone <your-repo-url>
cd mychat-app
npm install
```

### 2. Set Up Convex
```bash
npx convex dev
```
- Login or create a Convex account
- Create a new project
- This will generate `.env.local` with Convex URLs

### 3. Set Up Clerk
1. Go to [clerk.com](https://clerk.com) and create an account
2. Create a new application
3. Enable Email and Google OAuth providers
4. Copy your API keys

### 4. Configure Environment Variables
Create `.env.local` in the root directory:
```env
# Convex (auto-generated by `npx convex dev`)
CONVEX_DEPLOYMENT=<your-deployment>
NEXT_PUBLIC_CONVEX_URL=<your-convex-url>

# Clerk
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=<your-publishable-key>
CLERK_SECRET_KEY=<your-secret-key>
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
```

### 5. Run Development Server
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000) in your browser.

### 6. Deploy to Vercel
```bash
# Push to GitHub first
git add .
git commit -m "Initial commit"
git push origin main

# Deploy to Vercel
npx vercel
```
Add all environment variables in Vercel dashboard.

## 📁 Project Structure

```
mychat-app/
├── app/                    # Next.js app router
│   ├── layout.tsx         # Root layout with providers
│   └── page.tsx           # Main chat interface
├── components/            # React components
│   ├── ChatInterface.tsx  # Main chat layout
│   ├── UserList.tsx       # User discovery
│   ├── ConversationList.tsx # Conversation sidebar
│   └── ChatWindow.tsx     # Message display
├── convex/                # Backend code
│   ├── schema.ts          # Database schema
│   ├── mutations.ts       # Write operations
│   └── queries.ts         # Read operations
└── .env.local            # Environment variables
```

## 🗄️ Database Schema

### Tables
- **users** - User profiles with online status
- **conversations** - 1-on-1 and group chats
- **messages** - All messages with soft delete
- **typingIndicators** - Real-time typing status
- **reactions** - Message reactions
